金融系统中的智能提示设计：规则 + AI 的组合方式

背景

金融业务系统的提示信息通常承担三类职责：

  • 解释：解释当前输入/选择会导致什么结果
  • 提醒：提醒用户注意关键条件或缺失信息
  • 引导：引导用户完成复杂流程（多步骤、强校验）

仅用固定文案会遇到覆盖不足与维护困难；直接使用 AI 生成又会遇到不可控与一致性问题。更稳妥的方案是：

规则系统负责“判断与边界”，AI 负责“表达与可读性”。

设计目标

  • 输出内容可控：在可接受范围内变化
  • 结果可追溯：能够解释“为什么提示这句话”
  • 可回退：AI 不可用时仍能正常提示
  • 可治理：敏感场景与关键路径有明确禁用策略

架构分层：判断层与表达层解耦

推荐分层如下：

UI 层（表单/流程）
  ↓
提示编排层（Hint Orchestrator）
  ↓
判断层（Rules / Policy）
  ↓
表达层（AI 文案优化，可选）
  ↓
展示层（Toast/Alert/Inline）

判断层输出必须结构化

判断层不要直接输出最终中文文案，而应输出结构化结果，例如：

{
  "code": "MISSING_REQUIRED_FIELD",
  "severity": "high",
  "facts": {
    "field": "联系地址",
    "step": "基本信息"
  },
  "suggestions": [
    "补全联系地址后可继续下一步",
    "若地址与证件不一致，请按证件信息填写"
  ]
}

优势：

  • 可测试：可以对 code/severity/facts 写单元测试
  • 可统计：能够统计高频提示类型，反向优化流程
  • 可控：AI 仅消费这份结构化数据，不越权做判断

表达层：AI 的职责边界

AI 只做三件事：

  1. 将结构化信息组织成更易读的提示
  2. 统一语气与风格（短句、明确动作、避免模糊词）
  3. 生成多版本文案供选择（可选）

AI 不允许：

  • 引入新事实（例如编造规则、编造后果）
  • 改变严重级别（high/medium/low）
  • 改写关键数值/名词（产品名、字段名、阈值）

可控生成的关键做法

1) 模板主导 + AI 填充

先用模板把骨架固定：

【问题】{field} 未填写
【影响】无法继续 {step}
【建议】{suggestion_1}

再让 AI 做“表达优化”，例如将三行压成一段更自然的话，但不改变含义。

2) 输出长度与格式约束

给 AI 明确约束：

  • 字数上限（例如 60–90 字）
  • 禁止使用“可能/大概/建议考虑”等模糊词
  • 必须包含动作词（补全/修改/确认/返回）
  • 必须包含字段名（来自 facts，不允许改写）

3) 敏感词与口径过滤（工程兜底）

即使 AI 输出“看起来没问题”，仍建议做一层过滤：

  • 黑名单词（不应出现的表述）
  • 不允许出现“保证/承诺”等绝对化措辞
  • 不允许出现未在 facts 中出现的数值

过滤失败时：

  • 直接回退到模板文案
  • 记录日志用于改进 prompt/规则

交互形态选择：提示不是越多越好

提示信息常见展示方式：

  • Inline（字段下方）：适合即时校验
  • Banner（页内顶部）：适合步骤级提示
  • Modal（弹窗）：适合阻断式高风险提醒
  • Toast：适合轻提示，但不适合重要信息

建议用 severity 驱动展示策略，例如：

  • high：阻断式（Modal/Banner），必须处理
  • medium：Banner/Inline，建议处理
  • low：Inline/Toast，允许忽略

可靠性：超时、失败、降级

AI 相关链路建议默认具备：

  • 超时：例如 800ms–1500ms（按体验要求）
  • 失败降级：失败直接显示模板文案
  • 缓存：同一 code + facts 的结果可缓存 1–24h（按更新频率）

评测与治理：让效果“可量化”

最小评测集

对每个高频提示码，准备 5–10 个样例输入，覆盖：

  • 正常路径
  • 边界值
  • 异常输入
  • 缺失字段
  • 多字段联动

评估维度

  • 读完是否知道下一步做什么
  • 是否包含明确字段名与动作
  • 是否引入了规则之外的新信息
  • 是否保持与严重等级匹配的语气

总结

在金融系统中引入 AI 生成提示，关键不在“生成能力”，而在“工程可控”：

  • 规则负责判断与边界，AI 负责表达与可读性
  • 判断层输出结构化，表达层严格受限
  • 必须具备过滤、降级、日志、评测等治理能力

把 AI 放在可替换的增强层，才能在长期演进中保持稳定与安全。
